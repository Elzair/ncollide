Function ncollide::narrow::private::implicit_implicit::collide

pub fn collide<S: Simplex<N, AnnotatedPoint<V>>, G1: Implicit<N, V, M>, G2: Implicit<N, V, M>, N: std::ops::Sub + std::cmp::Ord + std::ops::Mul + std::num::Float + std::clone::Clone + std::to_str::ToStr, V: nalgebra::traits::structure::AlgebraicVecExt + std::clone::Clone + std::to_str::ToStr, M: nalgebra::traits::geometry::Translation + std::num::One>(m1: &M, g1: &G1, m2: &M, g2: &G2, prediction: &N, simplex: &mut S, init_dir: std::option::Option) -> std::option::Option

Computes a contact point between two implicit geometries. For optimizations purposes the objects are artificially enlarged by a small margin. This uses the GJK algorithm to handle penetrations within the margin and a Minkowski sum sampling based algorithm to handle deep penetrations.

Arguments:

  • g1 - the first implicit shape involved on the collision check
  • g2 - the second implicit shape involved on the collision check
  • simplex - the simplex the GJK algorithm must use. It is reinitialized before being passed to GJK.
