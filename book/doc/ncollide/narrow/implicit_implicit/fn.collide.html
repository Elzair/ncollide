Function ncollide::narrow::implicit_implicit::collide

pub fn collide<S: Simplex<N, AnnotatedPoint<V>>, G1: Implicit<N, V, M> + PreferedSamplingDirections<V, M>, G2: Implicit<N, V, M> + PreferedSamplingDirections<V, M>, N: Sub<N, N> + Ord + Mul<N, N> + Float + Cast<f32> + Clone, V: AlgebraicVecExt<N> + Clone, M: Translation<V> + One>(m1: &M, g1: &G1, m2: &M, g2: &G2, prediction: &N, simplex: &mut S, init_dir: Option<V>) -> GJKResult<Contact<N, V>, V>

Computes a contact point between two implicit geometries.

For optimizations purposes the objects are artificially enlarged by a small margin. This uses the GJK algorithm to handle penetrations within the margin and a Minkowski sum sampling based algorithm to handle deep penetrations.

Arguments:

  • g1 - the first implicit shape involved on the collision check
  • g2 - the second implicit shape involved on the collision check
  • simplex - the simplex the GJK algorithm must use. It is reinitialized before being passed to GJK.
